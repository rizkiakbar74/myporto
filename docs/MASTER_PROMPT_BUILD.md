# MASTER_PROMPT_BUILD — Prompt Implementasi Portfolio

Gunakan prompt ini jika ingin membangun project dengan AI coding tool. Jika dikerjakan manual, prompt ini tetap bisa menjadi spesifikasi implementasi.

```md
You are a senior frontend engineer and product designer.

Build a premium cinematic personal portfolio website for Muhammad Rizki Akbar using Bahasa Indonesia.

Goal:
Redesign the existing portfolio into a modern, cinematic, dark-themed portfolio inspired by AI developer portfolio videos, but keep the content credible and grounded.

Tech stack:
- Vite + React + TypeScript
- Tailwind CSS
- Framer Motion
- lucide-react

Design direction:
- Dark cinematic theme
- Blue/cyan/violet glow accents
- Subtle red/orange glow allowed
- Glassmorphism cards
- Smooth scroll reveal animation
- Large typography
- Premium project showcase
- Responsive mobile-first layout
- No horizontal overflow
- Bahasa Indonesia only

Page sections:
1. Navbar
2. Hero Cinematic
3. Tentang Saya
4. Pengalaman Saya
5. Projek Pilihan
6. Keahlian & Tools
7. Galeri Foto
8. Kontak CTA
9. Footer

Hero copy:
Headline:
"Membangun Website Modern yang Rapi, Cepat, dan Mudah Digunakan."

Subtitle:
"Saya adalah Muhammad Rizki Akbar, lulusan Sistem Informasi yang fokus membangun website responsif, tampilan modern, dan solusi digital yang mudah digunakan."

CTA:
- Lihat Projek
- Hubungi Saya

Current content basis:
- Education: S1 Sistem Informasi
- Experiences:
  - Institut Teknologi Indonesia — Februari 2023 - Sekarang
  - PT. Kreasi Nostra Mandiri / Sayurbox — Januari 2021 - Desember 2022
  - PT. Dom Pizza Indonesia / Domino's Pizza — Februari 2020 - April 2020
  - CV. Central Jaya Teknik — Maret 2019 - Juni 2019
  - PT. Federal International Finance — Maret 2017 - September 2017
- Projects:
  - Website Portfolio
  - Website Perusahaan
  - Website Kedai Kopi

Important rules:
- Do not use English UI copy.
- Do not use lorem ipsum.
- Do not invent fake achievements, fake metrics, or fake job titles.
- If a job title is unknown, write neutral descriptions or leave placeholders in data.
- Use data files for projects, experience, and skills.
- Use reusable components.
- Build must pass.
- Mobile responsiveness is mandatory.
```
